# [문서13] 알고리즘 완전 기초 — 모르면 안 되는 상식
> 강의에서 당연시하고 넘어간 기본기.

---

## 자료구조 한눈에
- 배열: 인덱스 O(1), 삽입/삭제 O(n). 연속 메모리.
- 연결 리스트: 접근 O(n), 삽입/삭제 O(1). 포인터.
- 스택: LIFO. 큐: FIFO. 덱: 양쪽.
- 해시: 평균 O(1) 조회. 트리: 계층. 힙: 최대/최소 빠르게. 그래프: 노드+간선.

## 복잡도 감각 (n=백만 기준)
- O(1): 즉시 / O(log n): ~20회 / O(n): 백만 / O(n log n): 2천만 / O(n²): 1조(느림) / O(2ⁿ): 불가능.
- 실무 규칙: n이 크면 O(n²) 이상은 위험. O(n log n) 이하를 목표.

## 그래프 표현
- 인접 행렬: V×V 배열. 간선 확인 O(1), 공간 O(V²). 밀집 그래프.
- 인접 리스트: 각 노드의 이웃 목록. 공간 O(V+E). 희소 그래프(대부분).

## 정렬 안정성/제자리
- 안정(stable): 같은 값 순서 유지. 병합=안정, 퀵/힙=불안정.
- 제자리(in-place): 추가 메모리 O(1). 퀵/힙=제자리, 병합=O(n) 추가.

## 트리 순회
- 전위(root→L→R) / 중위(L→root→R, BST면 정렬 순) / 후위(L→R→root) / 레벨(BFS).

## 비트/수학 기본
- 2진수, 비트 연산(AND/OR/XOR/shift). XOR로 중복 찾기, 짝 검출.
- 모듈러 연산(나머지), 큰 수는 mod로 관리. GCD(유클리드 호제법).

## 재귀 3요소
- 종료 조건(base case) / 자기 호출 / 문제 축소. 종료 조건 없으면 무한 재귀→스택 오버플로우.

## 완전탐색 vs 최적화
- 완전탐색(brute force): 모든 경우. 정확하지만 느림(작은 n).
- 최적화: 그리디/DP/분할정복으로 탐색 공간 축소.
- 순서: 일단 완전탐색으로 정답 확인 → 시간 초과면 최적화.

## 대표 문제 유형
- 정렬/탐색, 투 포인터, 슬라이딩 윈도우, 누적합, 이분 탐색,
- DFS/BFS, DP, 그리디, 백트래킹, 그래프(최단경로/MST/위상정렬).
