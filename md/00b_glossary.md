# [문서] 상식 수준 용어 체크리스트 — 한 줄로 설명 가능해야
> 경력직이면 "이 정도는 당연히" 나오는 기본 용어. 깊이 파는 게 아니라 들었을 때 바로 반응+한 줄 설명.
> 기준: 각 용어를 한 문장으로 설명할 수 있는가? 막히면 그 개념부터 복습.

---

## 🌐 네트워크
프로토콜·계층: TCP/UDP, IP, HTTP/HTTPS, HTTP/2·3, TLS/SSL, DNS, OSI 7계층, TCP/IP 4계층
TCP 동작: 3-way/4-way handshake, TIME_WAIT, 흐름 제어, 혼잡 제어, MTU/MSS
HTTP: 메서드(GET/POST/PUT/PATCH/DELETE), 상태코드(2xx/3xx/4xx/5xx), 헤더, 쿠키/세션, 멱등성, REST, stateless
보안·주소: CORS, CSRF, XSS, 방화벽, 포트, 공인/사설 IP, NAT, 프록시(포워드/리버스), 로드밸런서
응용: WebSocket, gRPC, GraphQL, CDN, VPN, ARP, 소켓
한 줄 테스트:
- TIME_WAIT → 연결 종료 후 잔여 패킷 처리 위해 잠깐 대기하는 상태.
- 멱등성 → 같은 요청 여러 번 보내도 결과 같음(GET/PUT 멱등, POST 아님).
- CORS → 다른 출처(프로토콜+도메인+포트) 요청을 브라우저가 막는 정책.
- 3-way handshake → SYN→SYN/ACK→ACK로 연결 수립.
- REST → 자원을 URL로, 행위를 HTTP 메서드로 표현하는 stateless API 스타일.

## 🗄️ 데이터베이스
인덱스·쿼리: 인덱스, B+Tree, 클러스터드/논클러스터드, 복합 인덱스, 커버링 인덱스, 카디널리티, 풀스캔, EXPLAIN(실행계획), 옵티마이저
트랜잭션: 트랜잭션, ACID, 커밋/롤백, 격리 수준(4종), 락(공유/배타), 데드락, MVCC, 낙관적/비관적 락
설계: 정규화(1~3NF), 반정규화, PK/FK, 참조 무결성, 스키마, 관계(1:1/1:N/N:N)
조인·기법: INNER/LEFT/RIGHT/OUTER JOIN, 서브쿼리, CTE(WITH), GROUP BY, HAVING, 집계함수, UNION, 손익분기점
운영·확장: 복제, 샤딩, 파티셔닝, 커넥션 풀, N+1, 슬로우 쿼리, WAL, 백업/복구
SQL vs NoSQL: 관계형/문서형/키-값, ACID vs BASE, 최종/강한 일관성
한 줄 테스트:
- 카디널리티 → 컬럼 값의 다양성. 높을수록 인덱스 효과 큼.
- N+1 → 목록 1번+각 행 연관 조회 N번으로 쿼리 폭증.
- MVCC → 락 대신 버전 스냅샷으로 읽기·쓰기 동시성 확보.
- 정규화 → 중복 제거 위해 테이블 나누고 FK로 연결.
- 샤딩 → 데이터를 여러 DB에 수평 분할(쓰기 확장).
- 커버링 인덱스 → 쿼리에 필요한 컬럼이 인덱스에 다 있어 테이블 안 봐도 됨.

## 🧮 알고리즘·자료구조
복잡도: 시간/공간 복잡도, Big-O, 최악/평균/최선, O(1)/O(log n)/O(n)/O(n log n)/O(n²)/O(2ⁿ)
자료구조: 배열, 연결 리스트, 스택(LIFO), 큐(FIFO), 덱, 해시테이블, 트리, BST, 힙, 그래프, 트라이
정렬·탐색: 버블/선택/삽입, 퀵, 병합, 힙 정렬, 안정 정렬, 이진 탐색, 선형 탐색
기법: 재귀, 분할 정복, DP, 그리디, 백트래킹, 완전 탐색, 투 포인터, 슬라이딩 윈도우, 메모이제이션
그래프: DFS, BFS, 다익스트라, 최단 경로, MST, 위상 정렬, Union-Find
해시·기타: 해시 함수, 해시 충돌, in-place, 재귀 vs 반복
한 줄 테스트:
- 스택 vs 큐 → 스택 LIFO(후입선출), 큐 FIFO(선입선출).
- DP → 겹치는 부분 문제를 저장해 재계산 없이 푸는 것.
- 이진 탐색 → 정렬된 배열을 반씩 줄여 O(log n)으로 찾기.
- 해시 충돌 → 다른 키가 같은 해시값. 체이닝/개방주소법으로 해결.
- DFS vs BFS → 깊이 우선(스택/재귀) vs 너비 우선(큐, 최단거리).
- 안정 정렬 → 같은 값의 원래 순서 유지(병합 O, 퀵 X).
- 그리디 → 매 순간 최선 선택. 항상 최적은 아님(반례 확인 필요).

## 활용법
- 각 용어를 한 줄로 설명해보고, 막히면 해당 분야 문서로.
- 면접에서 이 용어들은 "설명"이 아니라 "반응" 대상 — 나오면 즉시 알아듣고 한 줄로 받아치기.
- 깊이는 후속 질문에서. 여기 있는 건 "당연히 아는" 바닥 상식선.
