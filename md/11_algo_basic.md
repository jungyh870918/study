# [문서11] 자료구조·알고리즘 기본 — 면접 실전용
> 포맷: 용어 정의 → 핵심 개념(한두 줄) → 문제/인과관계
> 핵심은 암기가 아니라 "복잡도로 사고하고 상황에 맞는 자료구조를 고르는 것".

---

## 1. 시간/공간 복잡도 (Big-O)
정의: 입력 n이 커질 때 연산 횟수/메모리가 어떻게 증가하는지. 상수·낮은 차수 무시, 지배적 증가율만.
인과: n 충분히 크면 증가율이 성능 지배. n=100만이면 O(n) vs O(n²)는 100만 배. 상수 2배는 무의미.
증가율(느린→빠른):
- O(1) 상수: 배열 인덱스, 해시 조회
- O(log n): 이진 탐색, 균형 트리
- O(n): 배열 순회
- O(n log n): 효율적 정렬(병합/퀵/힙)
- O(n²): 이중 루프, 버블
- O(2ⁿ): 부분집합 완전탐색
- O(n!): 순열 완전탐색
★ 최악/평균/최선: 퀵정렬 평균 O(n log n)/최악 O(n²)(정렬된 데이터+나쁜 피벗). 해시 평균 O(1)/최악 O(n)(충돌). "평균만 말하면 반쪽, 최악을 알아야 안전."
★ 분할상환(amortized): 동적 배열 append 대부분 O(1), 가끔 O(n)(2배 확장). 평균 O(1).

## 2. 배열 vs 연결 리스트
정의: 둘 다 순차 데이터, 저장 방식 다름.
- 배열: 연속 메모리. 인덱스 접근 O(1), 중간 삽입/삭제 O(n), 캐시 친화적.
- 연결 리스트: 포인터 연결(흩어짐). 인덱스 접근 O(n), 삽입/삭제 O(1)(위치 알면), 포인터 오버헤드.
인과: 배열은 연속이라 인덱스 계산으로 바로 접근+캐시 유리 → 실무 기본(동적 배열). 연결 리스트는 삽입/삭제 잦고 위치 아는 특수 상황.

## 3. 스택 & 큐
- 스택(Stack): LIFO 후입선출. 함수 호출스택, undo, 괄호 검사, DFS.
- 큐(Queue): FIFO 선입선출. 작업 대기열, BFS, 메시지 큐.
- 덱(Deque): 양쪽 삽입/삭제.
- 우선순위 큐: 우선순위 순으로 꺼냄 → 보통 힙으로 구현.

## 4. 해시 테이블 (실무 최다)
정의: 키를 해시 함수로 인덱스 변환해 저장. 평균 O(1) 조회/삽입/삭제.
인과: 해시 함수가 인덱스 바로 계산 → 탐색 없이 접근.
- 충돌: 다른 키가 같은 인덱스 → 체이닝(연결 리스트)/개방 주소법(다음 빈자리).
- ★ 최악 O(n): 충돌 몰리면 한 버킷에 쌓여 순회. 좋은 해시 함수+부하율(load factor) 중요. 넘으면 리해싱(더 큰 배열 재배치).
실무: dict/map/HashMap. "빠른 조회=해시" 반사.

## 5. 트리 & 이진 탐색 트리(BST)
정의: 트리=계층(부모-자식). BST=왼쪽<부모<오른쪽 → 탐색 O(log n).
★ 균형 문제: 삽입 순서 나쁘면(정렬 데이터) 한쪽 치우쳐 O(n) 퇴화(연결 리스트化).
- 해결: 균형 트리 AVL/레드-블랙(자동 재균형, O(log n) 보장). Java TreeMap/C++ map=레드-블랙.
- B-Tree/B+Tree: 디스크용(DB 인덱스). 노드에 여러 키→낮고 넓음.

## 6. 힙(Heap)
정의: 부모가 자식보다 항상 크거나(최대) 작은(최소) 완전 이진 트리. 최대/최솟값 O(1) 확인, O(log n) 삽입/삭제.
용도: 우선순위 큐, Top-K(가장 큰/작은 K개), 힙 정렬, 다익스트라. "최댓값 계속 빠르게 꺼내면 힙" 반사.

## 7. 정렬 알고리즘
정의: 순서대로 나열. "최고"가 아니라 상황에 맞는 선택.
특성(5년차는 특성으로 기억):
- 버블/선택/삽입: O(n²). 작은 데이터·거의 정렬된 것(삽입정렬). 삽입만 안정.
- 병합(Merge): 평균/최악 다 O(n log n), 공간 O(n), 안정. 최악 보장·안정 필요할 때.
- 퀵(Quick): 평균 O(n log n)/최악 O(n²), 제자리·캐시 유리. 실무 기본.
- 힙(Heap): O(n log n) 보장+제자리, 상수 큼.
★ 포인트:
- 퀵이 실무 기본: 평균 빠름+제자리+캐시 친화. 최악 O(n²)(정렬 데이터+나쁜 피벗)→피벗 랜덤/중앙값으로 회피.
- 병합: 최악도 O(n log n)+안정. 단 O(n) 추가 메모리.
- 안정 정렬: 같은 값 원래 순서 유지("이름순 후 나이순"에서 이름 순서 유지 필요 시).
- 실무 정렬(Java/Python)=Timsort(병합+삽입 하이브리드, 안정, 거의 정렬된 데이터 빠름).

## 8. 이진 탐색
정의: 정렬된 배열에서 중간값 비교해 절반씩 줄임. O(log n).
인과: 매번 후보 절반 → 100만도 20번. 전제=정렬돼 있어야. 한 번 정렬 후 여러 번 탐색 시 유리.
★ Lower/Upper Bound: "이 값 이상이 처음 나오는 위치" 변형. 범위 쿼리·삽입 위치. off-by-one 자주 틀림.

## 9. 재귀 & 분할 정복
- 재귀: 함수가 자신 호출, 같은 형태 작은 문제로.
- 분할 정복: 반으로 나눠 각각 풀고 합침. 병합/퀵/이진탐색의 뼈대.
★ 주의:
- 스택 오버플로우: 너무 깊으면 호출 스택 넘침 → 반복문 변환/꼬리 재귀.
- 중복 계산: 순진한 피보나치=O(2ⁿ) 반복 → 메모이제이션/DP(3강).

## 10. 그래프 탐색 DFS & BFS
- DFS(깊이 우선): 끝까지 갔다 되돌아옴. 스택/재귀. 경로 존재, 사이클 검출, 위상 정렬.
- BFS(너비 우선): 가까운 것부터 층층이. 큐. 최단 경로(가중치 없을 때), 레벨 탐색.
★ 선택: "최단 거리(간선 수)"→BFS / "모든 경로·백트래킹"→DFS. 둘 다 O(V+E).

## 11. 백트래킹
정의: DFS로 모든 경우 탐색하되 가망 없으면 즉시 되돌아감(가지치기)로 공간 축소. 순열/조합, N-Queens, 스도쿠, 미로.
핵심: "해보고 안 되면 되돌린다." 완전 탐색이지만 pruning으로 실용적 속도.

## 12. 동적 계획법(DP)
정의: 큰 문제를 작은 부분 문제로 쪼개고 답을 저장해 재사용. 중복 계산 해결.
두 조건: 겹치는 부분 문제(같은 작은 문제 반복)+최적 부분 구조(큰 최적해가 작은 최적해로 구성).
구현: Top-down(메모이제이션, 재귀+캐싱) / Bottom-up(타뷸레이션, 표 채워 올라감, 스택오버플로우 없음).
★ 어려움=상태 정의+점화식 찾기. O(2ⁿ) 완전탐색을 O(n²)/O(n)으로. 예: 피보나치 O(2ⁿ)→O(n), 배낭, LCS, 편집 거리.

## 13. 그리디
정의: 매 순간 당장 최선 선택(지역 최적).
인과: 통하려면 지역 최적 연속이 전역 최적 되는 구조(그리디 선택 속성+최적 부분 구조).
- 안 통하는 예: 동전[1,3,4]로 6원 → 그리디 4+1+1=3개, 최적 3+3=2개. 이땐 DP.
★ 그리디 vs DP: 그리디 빠르지만 항상 옳지 않음, DP 느리지만 항상 최적. "그리디가 맞다는 증명"되면 그리디. 대표: 회의실 배정, 허프만, MST.

## 14. 최단 경로
- BFS: 가중치 없는 최단 경로. O(V+E).
- 다익스트라: 양의 가중치. 매번 가장 가까운 노드 확정(그리디)+우선순위 큐. O((V+E)log V). 내비.
- 벨만-포드: 음의 가중치 허용, 음수 사이클 검출. O(VE) 느림.
- 플로이드-워셜: 모든 쌍. O(V³). 노드 적을 때.
★ 선택: 가중치 없음→BFS, 양수→다익스트라, 음수→벨만-포드, 모든 쌍→플로이드. "음수면 다익스트라 못 씀"이 함정.

## 15. 최소 신장 트리(MST)
정의: 모든 노드를 최소 비용으로 연결(사이클 없이). 네트워크/도로.
- 크루스칼: 간선 가중치 순 정렬→사이클 안 생기게 추가. Union-Find로 검사. O(E log E).
- 프림: 한 노드서 시작해 가장 싼 간선으로 확장. 힙.

## 16. Union-Find (서로소 집합)
정의: 원소가 어느 그룹인지 빠르게 관리. "같은 그룹인가"+"합치기"를 거의 O(1).
핵심: 경로 압축+랭크로 거의 상수. 사이클 검출, 연결성, 그룹핑. "연결 관계·그룹"이면 반사.

## 17. 위상 정렬
정의: 방향 그래프에서 순서 제약 지키는 나열(선수→후수). 사이클 없는 DAG만.
용도: 작업 스케줄링, 빌드 의존성, 강의 선후. DFS 또는 진입차수(in-degree) BFS.

## 알고리즘 선택 치트 (실무 반사)
- 정렬된 데이터 찾기 → 이진 탐색
- 빠른 조회/중복 체크 → 해시
- 최댓값 계속 꺼내기/Top-K → 힙
- 최단 거리(가중치 없음) → BFS
- 최단 거리(양의 가중치) → 다익스트라
- 모든 경로/조합 → DFS/백트래킹
- "최적"인데 그리디 틀림 → DP
- 연결 관계/그룹 → Union-Find
- 순서 제약 나열 → 위상 정렬
- 최소 비용 연결 → MST
